01 · Ano ang MCP

Isang protocol. Anumang AI ay nakikipag-usap sa iyong account.

Ang MCP (Model Context Protocol) ay isang bukas na pamantayan mula sa Anthropic kung saan kumokonekta ang mga AI assistant sa mga panlabas na sistema. Nagho-host kami ng mga server para sa bawat uri ng instrumento — gagawa ka ng token at i-paste ito sa iyong assistant.

Claude

Ipakita ang aking balanse

Ang balance mo ay $2,480.50 — 2 bukas na posisyon (EUR/USD, BTC), $1,920 na libreng margin.

Ipakita sa akin ang EUR/USD quote

Ang EUR/USD ay nasa 1.0847, +0.12% ngayong araw.

Suriin ang mga market sa simpleng wika

Tanungin ang AI tungkol sa live na quotes, makasaysayang candles, spreads, volumes — sa sarili mong mga salita, sa lahat ng 4 na market nang sabay-sabay.

Tingnan ang iyong portfolio sa chat

Mga bukas na posisyon, mga nakabinbing order, kasaysayan ng trade, kabuuang equity — lahat sa pamamagitan ng simpleng pag-uusap. Walang paglipat ng window.

Kontrol sa antas ng token

Ang bawat token ay isang hiwalay na integrasyon na may sariling tagal ng buhay. Tingnan ang lahat ng aktibo sa Settings → AI integrations at bawiin ang alinman sa mga ito sa isang click.

03 · Mga pahintulot ng token

Read bilang default. Trade — opt-in, bawat market.

Bawat token ay may sariling saklaw. Ang Read access sa lahat ng 4 na server ay awtomatikong naka-enable at hindi maaaring patayin — kailangan ito ng AI para lang "makita" ang iyong data. Ang karapatang magbukas at magsara ng mga trade ay ipinagkakaloob nang tahasan — hiwalay para sa bawat market, kapag gumawa ka ng token.

Read — hindi maaaring patayin

Kung walang read access ang AI ay walang silbi — hindi nito maipapakita ang iyong balanse o mga presyo. Kaya naman ang read scope ay awtomatikong naka-enable sa lahat ng 4 na server. Ito ang pinakaligtas na antas: ang AI ay tumitingin lang, hindi kailanman humahawak.

Trade — eksakto, bawat use case

Gusto mo bang tulungan ka ng AI na mag-trade ng options lang? Lagyan ng tsek ang Trade · Digital Options at wala nang iba. Ang Margin CFD/Crypto/Forex ay mananatiling read-only — kahit subukan ng AI, ibabalik ng server ang 403 Forbidden.

Maaaring baguhin ang saklaw nang hindi muling gumagawa ng token

Gusto mo bang alisin ang pahintulot sa trade mula sa isang token na ginagamit na? Buksan ang Settings → AI integrations, hanapin ang token, alisin ang tsek sa market — agad na maaalis ang pahintulot. Ang halaga ng token mismo ay hindi nagbabago, kaya hindi mo na kailangang i-update ang config ng iyong assistant.

04 · Paano kumonekta

Tatlong hakbang. Walang code, walang terminal.

Ang aming MCP ay tumatakbo sa streamable-HTTP — walang Python environments, walang virtual venv, walang repos na ipa-clone. URL lang + Bearer token na gagawin mo sa isang click.

1

Gumawa ng token

Sa mobile app o sa web:

  1. Buksan ang Settings → AI integrations →
  2. I-tap ang + Add integration
  3. Bigyan ang token ng pangalan at pag-expire
  4. Lagyan ng tsek kung aling mga market ang pinapayagan para sa trading (ang read ay laging nakabukas). Mga detalye — seksyon 03 sa itaas
  5. Kopyahin agad ang token — kapag isinara mo ang screen ay hindi na ito ipapakita muli

06 · Kaligtasan

Iyong token, iyong responsibilidad. Naka-built in ang mga guardrail.

Ibinibigay ng MCP sa AI ang mga susi sa iyong trading account. Kaya kami — at ikaw — ay sabay na nagbabantay sa mga limitasyon.

Nakatira ang token sa headers, hindi sa chat

Ang Bearer token ay nakakabit sa HTTP header sa koneksyon ng assistant ↔ server. Hindi ito nakikita ng AI sa katawan ng mensahe at hindi ito maaaring aksidenteng maipadala palabas.

Huwag ibahagi. Huwag i-commit.

Ang token = access sa iyong trading account. Huwag kailanman i-publish sa git, screenshots, chats o gists. Kung pinaghihinalaan mong may leak — buksan ang Settings → AI integrations at pindutin ang Revoke sa tabi ng token na iyon. Lahat ng kahilingang gumagamit nito ay agad na titigil sa paggana.

Pag-expire at pag-rotate

Itinatakda mo ang pag-expire sa paggawa. Pagkatapos ng petsang iyon ay awtomatikong titigil sa paggana ang token. Inirerekomenda naming magpanatili ng hiwalay na mga token bawat integrasyon — sa ganoong paraan ay maaari mong bawiin lang ang kailangan mo.

Magsimula nang maliit

Makapangyarihan ang AI ngunit hindi walang pagkakamali. Maaari itong pumili ng maling instrumento, sukat, o direksyon. Bago ipagkatiwala sa model ang sukat — dumaan sa mga senaryo nang manu-mano at suriing mabuti ang mga mungkahi nito. Sa iyo ang responsibilidad para sa bawat trade.

08 · Mga madalas itanong

FAQ

Kailangan ko bang mag-install ng kahit ano sa aking computer?

Hindi. Ang aming mga MCP server ay naka-host namin — walang repos na ipa-clone, walang Python o venv na ia-install. URL lang + Bearer token sa config ng iyong AI assistant.

Talaga bang gumagana ang isang token sa lahat ng 4 na server?

Oo. Gumawa ng isang Bearer token sa Settings → AI integrations at gamitin ito sa Authorization header para sa lahat ng apat na endpoint: digital-options, marginal-cfd, marginal-crypto, marginal-forex.

Paano kung gusto kong ikonekta ang isang market lang, hal. Forex?

Burahin ang ibang mga block mula sa config — panatilihin lang ang iq-margin-forex. Makikita lang ng AI ang mga tool mula sa server na iyon at hindi nito mahahawakan, hal., ang options nang aksidente.

Ano talaga ang magagawa ng AI sa pamamagitan ng MCP?

Bawat isa sa aming mga server ay naglalantad ng sarili nitong listahan ng mga tool — makikita mo ang buong set kaagad pagkatapos kumonekta sa pamamagitan ng pagtatanong sa assistant: "Anong mga IQ Option tool ang mayroon ka?". Sa kabuuan ito ay pagbabasa ng market data, portfolio at trades, kasama ang mga paraan para magbukas/magsara ng mga posisyon. Aling mga paraan ang available sa isang partikular na token ay tinutukoy ng saklaw nito (read + opsyonal na trade bawat market).

Paano ko sisiguraduhin na hindi makapag-trade ang AI?

Kapag gumagawa ng token sa Settings → AI integrations ay basta huwag lagyan ng tsek ang alinman sa apat na "Trade · …" na checkbox. Ang Read ay nananatiling nakabukas (hindi ito maaaring patayin — kailangan ito ng AI), at anumang pagtatangkang tumawag sa isang trading tool ay magbabalik ng 403 Forbidden. Ito ang pinakaligtas na setup para sa mga use case ng pananaliksik.

Maaari ko bang payagan ang trading sa isang instrumento lang?

Oo. Halimbawa, lagyan ng tsek lang ang Trade · Margin Crypto — magagawa ng AI na magbukas/magsara ng mga crypto position, ngunit ang options, CFD at forex ay nananatiling read-only. Kapaki-pakinabang kapag sinusubok mo ang AI setup sa isang market na komportable ka.

Paano kung gusto kong alisin ang pahintulot sa trade mula sa isang umiiral na token?

Buksan ang Settings → AI integrations, hanapin ang token at alisin ang tsek sa kaugnay na box (hal. Trade · Margin CFD). Agad na maaalis ang pahintulot — magsisimula ang server na magbalik ng 403 Forbidden sa mga trading call para sa market na iyon. Ang halaga ng token mismo ay hindi nagbabago, kaya hindi mo na kailangang hawakan ang config ng iyong assistant.

Paano kung nag-leak ang aking token?

Pumunta sa Settings → AI integrations, hanapin ang nakompromisong token at pindutin ang Revoke. Lahat ng kahilingang gumagamit nito ay agad na titigil sa paggana. Gumawa ng bago at i-update ang config ng assistant.

Ano ang mga limitasyon sa mga token at server?

Ang isang account ay maaaring magkaroon ng maraming aktibong token nang sabay-sabay (nakikita sa listahan ng Active tokens). Isang token ang gumagana sa lahat ng 4 na server. Kung gusto mo ng hiwalay na mga key bawat integrasyon para sa mas madaling pagbawi — gumawa ng ilan at bigyan sila ng makabuluhang pangalan.

Hindi nakikita ng aking assistant ang mga tool pagkatapos ng setup. Ano na?

Checklist: (1) i-restart nang buo ang assistant, (2) tiyakin na ang token ay nakopya nang walang labis na whitespace sa simula/dulo, (3) suriin na ang URL ay walang trailing slash at tumutugma sa iyong subscription, (4) tiyakin na ang Authorization header ay nagsisimula nang eksakto sa Bearer (pansinin ang espasyo), (5) suriin ang mga log ng assistant — kadalasan ito ay JSON syntax error sa config.

Aling assistant ang pinakamadaling i-set up mula sa simula?

Kung hindi ka developer — Claude Desktop o Cherry Studio: pareho silang may UI na humahawak sa config para sa iyo. Kung nabubuhay ka sa code — mas magiging natural ang Cursor o Claude Code.
